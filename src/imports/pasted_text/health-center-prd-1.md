# ١ — نظرة عامة على المشروع





## ١.١ وصف المشروع



نظام إدارة المركز الصحي هو تطبيق ويب متكامل موجَّه للإدارة الداخلية لمركز صحي يضم أقساماً متعددة. يُتيح النظام للمستخدم الوحيد (الأدمن) إدارة جميع عمليات المركز من خلال لوحة تحكم موحَّدة باللغة العربية. يشمل النظام إدارة ملفات المرضى، الجلسات الطبية، العمليات المالية، الرواتب، المخزون، والتقارير الشاملة.



## ١.٢ المستخدم المستهدف



| **المستخدم الوحيد: الأدمن (مدير المركز)** |

| --- |

| • صلاحية كاملة على جميع أقسام النظام • إدارة الموظفين وتعديل صلاحياتهم • الوصول لجميع التقارير المالية والإحصائية • إعدادات النظام الكاملة والنسخ الاحتياطي |

# ٤ — الأقسام الطبية — Medical Departments





يشترك كل قسم طبي في هيكل موحَّد من ٣ أقسام فرعية. هذا القسم يوثِّق البنية المشتركة، ثم الخصائص الفريدة لكل قسم.



## ٤.١ الهيكل الموحَّد لكل قسم طبي



| **القسم الفرعي** | **المحتوى** |

| --- | --- |

| **١. فتح ملف مريض** | تسجيل مريض جديد + بدء جلسة │ أو بدء جلسة لمريض مسجل مسبقاً |

| **٢. الرئيسية والإحصائيات** | الجرار (صندوق الكاش) + ملخص القسم + قائمة الموظفين |

| **٣. المصروفات وفاتورة شراء** | المصروفات الشخصية + فواتير الشراء من الشركات |



## ٤.٢ تفاصيل قسم فتح ملف مريض — تسجيل مريض جديد



**عند اختيار تسجيل مريض جديد، يظهر نموذج مع الحقول التالية:**



| **اسم الحقل** | **إلزامي / اختياري** | **ملاحظات التصميم** |

| --- | --- | --- |

| **اسم المريض الكامل** | إلزامي | حقل نصي — Placeholder: أدخل الاسم الرباعي |

| **رقم الهوية** | إلزامي | حقل رقمي — يُستخدم كرقم ملف المريض |

| **رقم الجوال** | إلزامي | حقل هاتف — بادئة +970 |

| **تاريخ الميلاد** | إلزامي | Date Picker — يحسب العمر تلقائياً |

| **البريد الإلكتروني** | اختياري | حقل Email |

| **العنوان** | اختياري | حقل نصي |

| **الجنس** | إلزامي | Radio Buttons: ذكر / أنثى |

| **فصيلة الدم** | إلزامي | Dropdown: A+, A-, B+, B-, AB+, AB-, O+, O- |

| **حساسية / موانع دوائية** | إلزامي | Toggle: نعم / لا — إذا نعم: حقل نصي لإدخال التفاصيل |

| **أمراض مزمنة أو سابقة** | إلزامي | Toggle: نعم / لا — إذا نعم: حقل نصي للتفاصيل |

| **تأمين صحي** | إلزامي | Toggle: نعم / لا — إذا نعم: Dropdown بشركات التأمين المُدرجة |

| **ملاحظات إضافية** | اختياري | Textarea |

| **سعر الجلسة (₪)** | إلزامي | حقل رقمي — بالشيكل |

| **الخصم (₪ أو %)** | اختياري | حقل رقمي مع اختيار نوع الخصم |

| **المبلغ المدفوع (₪)** | إلزامي | حقل رقمي |

| **الباقي / الدين (₪)** | محسوب تلقائياً | يُحسب: السعر - الخصم - المدفوع — يُحفظ كدين على المريض |



## ٤.٣ ملف المريض



| **هيكل ملف المريض** |

| --- |

| • رقم الملف = رقم هوية المريض (ID بالنظام) • يحتوي على: معلومات المريض الأساسية، قائمة بجميع الجلسات مرتبة تاريخياً • النقر على أي جلسة يُظهر التفاصيل الكاملة للجلسة والتفاصيل المالية • للطبيب فقط إمكانية فتح تفاصيل الجلسة الطبية الكاملة |



## ٤.٤ ملف الجلسة — عيادة الجراحة العامة وعلاج تأهيلي



بعد بدء الجلسة وإدخال التفاصيل المالية، يُفتح نموذج ملف الجلسة:



| **التشخيص** | Autocomplete Dropdown — من التشخيصات المُدرجة مسبقاً أو إضافة تشخيص جديد |

| --- | --- |

| **وصف الجلسة** | Textarea RTL — وصف مفصَّل للجلسة |

| **الأدوية المُعطاة** | Multi-select من الأدوية المُدرجة مسبقاً أو إضافة دواء جديد |

| **المرفقات** | File Upload — صور، ملفات PDF، مستندات |



## ٤.٥ ملف الجلسة — مختبر التحاليل الطبية



خاصية فريدة: إدارة الفحوصات عبر مراحل:



| **المرحلة** | **الوصف** |

| --- | --- |

| **١. اختيار الفحوصات** | Multi-select من دليل الفحوصات مع الأسعار — ممكن أكثر من فحص |

| **٢. وصف الجلسة** | Textarea لوصف الحالة |

| **٣. الفحوصات قيد الإنجاز** | قائمة Status Board تظهر الفحوصات المنتظرة |

| **٤. إدخال النتائج** | عند الإنجاز: زر 'إدخال النتائج' → نموذج النتائج والكيتات |

| **٥. طباعة التقرير** | طباعة نتائج الفحص ضمن نموذج الطباعة الرسمي |

| **٦. تسليم التقرير** | زر 'تم تسليم التقرير' → تنتقل للفحوصات المُنجزة |



## ٤.٦ ملف الجلسة — الأشعة التشخيصية



خاصية فريدة: إدارة التصوير عبر مراحل:



| **اختيار الصور** | Multi-select من دليل الصور والأسعار — أكثر من صورة ممكن |

| --- | --- |

| **وصف جلسة التصوير** | Textarea |

| **التصوير قيد الإنجاز** | Status Board للجلسات المنتظرة |

| **رفع النتائج** | File Upload — ملفات الأشعة (JPEG/PNG/DICOM) |

| **تسليم النتائج** | زر 'تم تسليم النتائج' → تنتقل للجلسات المُنجزة |



## ٤.٧ الرئيسية والإحصائيات — مشترك لجميع الأقسام



| **📦 الجرار — صندوق الكاش (Cash Drawer)** |

| --- |

| • عنصر ثابت في أعلى الصفحة — يظهر دائماً • يعرض: النقد الحالي داخل الجرار (₪) • زر 'سحب نقد من الجرار': لتسجيل عملية إيداع في البنك — لا يُحتسب كمصروف • زر 'إضافة نقد للجرار': يُعامَل كـ 'سند قبض' مع إمكانية إدخال عنوان ثانوي |



**ملخص القسم يحتوي على:**



- إجمالي عدد المرضى في القسم



- إجمالي الإيرادات من القسم (₪)



- إجمالي المصاريف الشخصية للموظفين داخل القسم



- قائمة الموظفين داخل القسم مع أسمائهم وزر تعديل صلاحية كل موظف



## ٤.٨ المصروفات وفاتورة الشراء — مشترك لجميع الأقسام



**المصروفات الشخصية:**



- إنشاء مصروف شخصي: عنوان المصروف + المبلغ (₪)



- عرض جدول بتفاصيل المصروفات السابقة مع التواريخ



**فواتير الشراء:**



- إنشاء فاتورة شراء: جدول المنتجات (الاسم + الكمية + السعر)



- التفاصيل المالية: المجموع، الخصم من الشركة، المبلغ المدفوع، الباقي (دين للشركة)



- عرض جدول بفواتير الشراء السابقة



- زر 'طباعة' لكل فاتورة



- زر 'تسديد' للفواتير غير المسددة بالكامل — مع حقل إدخال المبلغ المُسدَّد (جزئي أو كامل)



## ٤.٩ الأقسام الفريدة الإضافية



| **القسم** | **القسم الفرعي الإضافي الفريد** |

| --- | --- |

| **مختبر التحاليل الطبية** | قسم دليل الفحوصات والأسعار: إضافة/تعديل/حذف الفحوصات مع تفاصيلها وأسعارها — استناداً لجدول Excel |

| **الأشعة التشخيصية** | قسم دليل الصور والأسعار: إضافة/تعديل/حذف أنواع الصور الشعاعية مع أسعارها |

# ٥ — النظام المالي للمركز — Financial System





النظام المالي هو القسم المركزي الذي يجمع كافة البيانات المالية من جميع الأقسام. يتكون من قسمين رئيسيين: المالية والكشوفات.



## ٥.١ قسم المالية — ٦ تبويبات



**التبويب الأول: الملخص المالي**



| 📊 | الإيرادات الكلية │ المصروفات الشخصية الكلية │ فواتير الشراء الكلية │ الديون الكلية على المرضى الديون الكلية على المرضى لكل قسم (رسم بياني) │ ديون للشركات │ ديون للشركات لكل قسم النقد الكلي داخل الجرار │ النقد الكلي المسحوب │ صافي الربح الإيرادات بحسب الأقسام (Chart) │ المصاريف الشخصية بحسب الأقسام (Chart) فواتير الشراء بحسب الأقسام (Chart) │ الربح حسب الأقسام (Chart) **⭐ اختيار أي فترة زمنية للكل دفعة واحدة + تصدير Excel أو PDF** |

| --- | --- |



**التبويب الثاني: الجرار**



| 💵 | كافة التفاصيل المتعلقة بحركات الجرار: الإيداعات، السحوبات، التواريخ، المبالغ رسوم بيانية توضح حركة النقد عبر الزمن │ اختيار فترة زمنية مخصصة تصدير كـ Excel أو PDF |

| --- | --- |



**التبويب الثالث: الإيرادات**



| 💰 | إيرادات كلية │ إيرادات حسب القسم │ إيرادات حسب الأشهر اختيار الفترة الزمنية بحرية تامة لكل عرض │ تصدير كافة البيانات |

| --- | --- |



**التبويب الرابع: المصروفات وفواتير الشركاء**



| 🧾 | إجمالي المصاريف الشخصية │ إجمالي فواتير الشراء المصاريف الشخصية بحسب كل قسم │ فواتير الشراء بحسب كل قسم **⭐ المصاريف الشخصية بحسب كل موظف — تُخصم من راتبه لاحقاً** فواتير الشراء حسب كل شركة │ إجمالي المبلغ الغير مسدد للفواتير **⭐ الفواتير غير المسددة بالكامل مع زر ****'****إجراءات السداد****'**** على كل فاتورة** |

| --- | --- |



**التبويب الخامس: الرواتب**



| 👨‍💼 | إجمالي رواتب الشهر │ إجمالي رواتب حسب الفترة الزمنية رسم بياني لرواتب الأقسام جدول الموظفين مع: — زر 'احتساب الراتب': يحسب الراتب الصافي بعد خصم المصروفات الشخصية — زر 'صرف الراتب لشهر [الشهر الحالي]': تأكيد صرف الراتب |

| --- | --- |



**التبويب السادس: الديون**



| ⚠️ | إجمالي الديون على المرضى │ الديون حسب القسم │ الديون حسب مدة التأخير **⭐ تفاصيل ديون المرضى + زر ****'****إرسال رسالة تذكير SMS****'**** على رقم المريض** اختيار الفترات بحرية │ حفظ تقارير PDF و Excel |

| --- | --- |



## ٥.٢ قسم الكشوفات — ٥ تبويبات

# ٦ — إعدادات النظام — System Settings





## ٦.١ تبويب أداة المخزون والكيتات



إدارة مخزون الكيتات والمواد المخبرية المرتبطة بالفحوصات.



| **اسم الصنف / Kit** | حقل نصي — مثال: CBC Kit |

| --- | --- |

| **الفحوصات المرتبطة** | Multi-select — ربط الفحوصات بالـ Kit (مثال: CBC → CBC Kit) |

| **آلية الاستهلاك** | كل طلب فحص مرتبط → يُخصم من الكمية الحالية بمقدار 1 تلقائياً |

| **الكمية الحالية** | عرض الكمية المتبقية + زر 'إضافة كمية' (يُعامَل كفاتورة شراء) |

| **حد التنبيه** | إدخال رقمي — عند الوصول للحد → إشعار داخل لوحة الأدمن وقسم المختبر |

| **الإشعار** | Notification Bell + رسالة: 'مخزون Kit [الاسم] وصل للحد الأدنى: [الكمية]' |

# ١٣ — النظام المالي المفصَّل — Financial System Deep-Dive





## ١٣.١ منطق حساب الجرار — Cash Drawer Logic



| **العملية** | **نوع السجل** | **التأثير على الجرار** |

| --- | --- | --- |

| **استلام مبلغ من مريض** | إيراد — دفعة جلسة | **+ يزيد رصيد الجرار** |

| **استلام مبلغ من مريض (دين)** | استلام دين سابق | **+ يزيد رصيد الجرار** |

| **إضافة نقد يدوية (سند قبض)** | سند قبض + عنوان ثانوي | **+ يزيد رصيد الجرار** |

| **سحب نقد (إيداع بنكي)** | حركة داخلية — ليس مصروفاً | **- يُنقص رصيد الجرار** |

| **مصروف شخصي لموظف** | مصروف — لا يمر بالجرار | **لا يؤثر على الجرار** |

| **دفع فاتورة شراء** | ديون شركات — لا يمر بالجرار | **لا يؤثر على الجرار** |



## ١٣.٢ معادلات حساب الإجماليات



| **الحقل المحسوب** | **المعادلة** |

| --- | --- |

| **دين جلسة المريض** | الدين = السعر − الخصم − المدفوع |

| **راتب الموظف الصافي** | الصافي = الراتب الأساسي − إجمالي مصروفاته الشخصية للشهر |

| **صافي ربح القسم** | الربح = إيرادات القسم − مصاريف القسم الشخصية − فواتير شراء القسم |

| **صافي ربح المركز** | الربح = إجمالي الإيرادات − إجمالي المصاريف الشخصية − إجمالي فواتير الشراء |

| **رصيد الجرار الحالي** | الرصيد = إجمالي الإيداعات − إجمالي السحوبات |

| **إجمالي ديون المرضى** | Σ (باقي كل جلسة لم يُسدَّد بالكامل) |

| **إجمالي ديون الشركات** | Σ (باقي كل فاتورة شراء لم تُسدَّد) |

| **مجموع فاتورة الشراء** | المجموع = Σ (الكمية × السعر) − خصم الشركة |

| **إجمالي إيرادات القسم** | Σ المبالغ المدفوعة في جلسات القسم (لا تشمل الديون غير المسددة) |



## ١٣.٣ التبويب ٤: المصروفات المفصَّل



| **القسم الفرعي أ** | المصاريف الشخصية: إجمالي + تفصيل بالموظف + تفصيل بالقسم |

| --- | --- |

| **القسم الفرعي ب** | فواتير الشراء: إجمالي + تفصيل بالشركة + تفصيل بالقسم |

| **جدول الموظفين** | الاسم │ القسم │ إجمالي مصاريفه │ تأثير على الراتب |

| **جدول الشركات** | اسم الشركة │ إجمالي الفواتير │ المدفوع │ الباقي الديون |

| **فواتير غير مسددة** | جدول منفصل: رقم الفاتورة │ الشركة │ القسم │ المبلغ │ الباقي │ زر السداد |

| **المبلغ الكلي غير المسدد** | KPI Card بلون أحمر خفيف — تنبيه بصري |

| **فلترة الفترة** | Date Range Picker يؤثر على الكل |

| **التصدير** | Excel + PDF منفصل لكل قسم فرعي |



## ١٣.٤ تبويب الكشوفات — هيكل التقارير المخصصة



| ⭐ | التقرير المخصص هو الأهم — يُتيح للمدير بناء أي تقرير يحتاجه من كل البيانات المتاحة |

| --- | --- |



| **الخطوة** | **الوصف** |

| --- | --- |
