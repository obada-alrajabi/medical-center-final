🚫 [🚨 أمر سيادي حاسم - الفصل الميكانيكي للملفات المالية بين الأقسام وقصرها على المدير]:
مرحباً Claude. لدينا تحديث أمني وحرج جداً يتعلق بخصوصية وحجب البيانات المالية للمرضى بين الموظفين في مختلف الأقسام (Financial Segregation & Role-Gated Visibility).

⚠️ تنبيه أمني برامجي صارم للغاية: يُمنع منعاً باتاً، وتحت أي مسمى أو ظرف، التعديل أو التغيير أو المساس بهيكلية المختبر الجديدة، المعادلات المالية الموحدة، شجرة الصلاحيات الهرمية، أو شاشة المذكرات المصلحة سابقاً. التعديل محصور "حصرًا" في شروط عرض البيانات المالية داخل ملف المريض (Patient File / Financial Component Gate).

يرجى تعديل شاشة ملف المريض ودوال جلب البيانات وفق المحددات التالية بدقة 100%:

### 🛑 القاعدة الأساسية للنظام (Unified Medical File / Isolated Financial File):
- المريض يمتلك ملفاً طبياً واحداً موحداً ومستقراً عبر النظام بأكمله لطلب الخدمات من عدة أقسام (العيادة، المختبر، الأشعة، التأهيل).
- يجب فصل وحجب السجلات والمعاملات المالية (الفواتير، المبالغ المدفوعة، المتبقية، السندات) بين الأقسام للموظفين، وقصر الرؤية الكاملة للشاشات المالية على مدير النظام (Admin).

### 🛠️ المطلوب برمجياً لتفعيل حجب البيانات (Granular Visibility Logic):

1. **🩺 في قسم عيادة الجراحة (حساب موظف العيادة):**
   - عند فتح ملف المريض، يرى موظف العيادة بيانات المريض الشخصية والطبية، بالإضافة إلى **المعاملات المالية الخاصة بقسم العيادة فقط لا غير**. 
   - يُحجب ويُمنع منعاً باتاً ظهور أي فواتير أو حركات مالية صادر من قسم المختبر أو الأشعة أو التأهيل أمام موظف العيادة.

2. **🧪 في قسم المختبر (حساب موظف المختبر):**
   - عند فتح ملف المريض، يرى موظف المختبر بيانات المريض والفحوصات، بالإضافة إلى **المعاملات المالية الخاصة بالفحوصات والمختبر فقط لا غير**.
   - يُحجب ويُمنع منعاً باتاً ظهور فواتير العيادة، ديون الجراحة، أو معاملات الأقسام الأخرى أمام موظف المختبر. وينطبق هذا الفصل والحجب الميكانيكي على بقية الأقسام (الأشعة والتأهيل) بالتناظر.

3. **👑 في حساب مدير النظام (Admin View - الصلاحية المطلقة):**
   - عند دخول المدير إلى ملف المريض من أي شاشة أو قسم، يتم كسر كافة القيود وتفعيل العرض الشامل التلقائي؛ فيظهر للمدير ملف المريض كاملاً بنسبة 100% يشمل: (البيانات الشخصية، التاريخ الطبي، التشخيصات في كل الأقسام، وكافة المعاملات والفواتير والديون المالية التراكمية لكل الأقسام مجتمعة).

4. **🔒 حماية الـ API والاستعلامات (Backend Controller Security):**
   - تأكد في دالة استدعاء البيانات المادية للمريض من فلترة المصفوفة بناءً على الـ Role والـ Department ID الخاص بالموظف الحالي لتجنب تسريب البيانات في الـ Network Tools.

⚠️ تأكيد ختامي ملزم: تقيد بالعمل البرمجي الحرفي لفصل وعزل البيانات المالية للمرضى بين حسابات الموظفين وحصر الرؤية الشاملة للمدير، وأبقِ كل شيء آخر في النظام على حاله بنسبة 100% دون أي تغيير، واعرض لي الأكواد المعدلة فوراً.
