PART 1 — SESSION REGISTRATION IS DIFFERENT PER DEPARTMENT (critical fix)
================================================================================

There are 3 DIFFERENT registration flows — not one shared flow:

  Flow A → عيادة الجراحة العامة والطوارئ  (current flow — keep as is)
  Flow B → مختبر التحاليل الطبية           (SELECT TESTS → auto-total)
  Flow C → الأشعة التشخيصية               (SELECT IMAGES → auto-total)
  Flow D → عيادة العلاج التأهيلي           (SAME AS Flow A — keep as is)

--------------------------------------------------------------------------------
FLOW A & D — عيادة الجراحة / العلاج التأهيلي (no change, keep existing)
--------------------------------------------------------------------------------
  Step 1: Patient data (new) OR patient search (existing)
  Step 2: Financial details — سعر الكشف + خصم + مدفوع → باقي
  Step 3: Session file — diagnosis + description + medications + attachments
  (This is already designed — DO NOT change it)

--------------------------------------------------------------------------------
FLOW B — مختبر التحاليل الطبية (completely different)
--------------------------------------------------------------------------------

ENTRY: مختبر التحاليل → فتح ملف مريض → واجهة الاختيار الأولى:
  Card A: [+ تسجيل مريض جديد]       → STEP 1-NEW
  Card B: [🔍 بدء جلسة لمريض مسجل]  → STEP 1-EXISTING

━━━ STEP 1-NEW: بيانات المريض الجديد ━━━
  Stepper: [① بيانات المريض] → [② اختيار الفحوصات] → [③ التفاصيل المالية]
  Current: ①
  Form: same patient data fields as other departments
  [التالي →] → STEP 2

━━━ STEP 1-EXISTING: بحث عن مريض مسجل ━━━
  Search by name or ID
  Select patient → patient summary card shown
  [تأكيد والمتابعة] → jumps directly to STEP 2 (skip patient data form)

━━━ STEP 2: اختيار الفحوصات المطلوبة ━━━
  Stepper: ② highlighted

  Layout: 2 columns
  Right column (60%): Test Selection

      Multi-select allowed (any number of tests)

  Left column (40%): Selected Tests Summary (sticky)

      [× CBC — ₪ 45]  [× TSH — ₪ 75]  [× LFT — ₪ 85]

  Kit quantities: NOT deducted yet (deducted when results are entered)

PART 2 — FINANCIAL DETAILS FIX (existing patient session)
================================================================================

PROBLEM: When starting a session for an existing patient, only price + paid
are shown. Discount and remaining debt are missing.

FIX: For BOTH new and existing patients, the financial step MUST ALWAYS show:

FOR CLINIC (Flow A/D) — existing patient:
  ┌─────────────────────────────────────────────┐
  │ سعر الكشف / الخدمة (₪) *        [____]    │
  │ الخصم (₪ أو %)                   [____]    │ ← was missing
  │ المبلغ المدفوع (₪) *             [____]    │
  │ ──────────────────────────────────────────  │
  │ الباقي من هذه الجلسة:           ₪ XX      │ ← was missing
  │   إن > 0: "سيُضاف لديون المريض"            │
  │                                             │
  │ إجمالي الديون السابقة:          ₪ YY      │ ← NEW: show existing debt
  │ إجمالي الدين بعد هذه الجلسة:   ₪ XX+YY   │ ← NEW: cumulative total
  └─────────────────────────────────────────────┘

FOR LAB (Flow B) — existing patient STEP 3:
  ┌─────────────────────────────────────────────┐
  │ المجموع (من الفحوصات) (₪)        ₪ 205   │ ← read-only, auto
  │ الخصم (₪ أو %)                   [____]   │
  │ المبلغ المدفوع (₪) *             [____]   │
  │ ──────────────────────────────────────────  │
  │ الباقي من هذه الجلسة:           ₪ XX      │ ← auto-calculated
  │                                             │
  │ ديون سابقة على المريض:          ₪ YY      │ ← from previous sessions
  │ إجمالي الدين الكلي بعد الجلسة:  ₪ XX+YY   │ ← cumulative
  └─────────────────────────────────────────────┘

FOR RADIOLOGY (Flow C) — existing patient STEP 3:
  Same structure as Lab above.

VISUAL TREATMENT for existing debt warning:
  If patient has existing debt > 0:
    Show amber banner above financial form:
    ┌──────────────────────────────────────────────────────┐
    │ ⚠️ هذا المريض لديه دين سابق بقيمة ₪ [YY]           │
    │    تاريخ أول دين: 15/03/2026                         │
    │    [عرض تفاصيل الديون] link → opens drawer          │
    └──────────────────────────────────────────────────────┘

DEBT SUMMARY LOGIC:
  الباقي من هذه الجلسة = المجموع - الخصم - المدفوع
  إجمالي الدين الكلي = الباقي من هذه الجلسة + مجموع ديون الجلسات السابقة

Each statement tab in قسم الكشوفات has its own UNIQUE layout:

━━━ TAB 1: كشوفات المرضى ━━━
  FILTER PANEL:
    Row 1: 🔍 ابحث عن مريض (by name or ID) — autocomplete

PART 4 — LAB TEST CATALOG: NEW REQUIRED FIELDS WHEN ADDING/EDITING
================================================================================

WHEN ADDING A NEW TEST, required fields are:

━━━ MODAL: إضافة فحص / تعديل فحص ━━━
  BASIC INFO section:
    اسم الفحص بالعربي *           text input
    اسم الفحص بالإنجليزي         text input
    رمز الفحص (Code) *            text input — e.g. CBC
    الفئة *                       dropdown (blood / urine / hormones / etc.)

  PRICING section (NEW — different from before):
    السعر للمريض (₪) *            number input — what patient pays
    سعر Lab to Lab (₪) *          number input — inter-lab / wholesale price
      Note under field: "السعر عند إرسال الفحص لمختبر خارجي"
    هامش الربح:                   auto-calculated, read-only
      Shows: "₪ X (X%)" — updates live as prices are typed

  INVENTORY / KIT section (NEW):
    اسم الكيت (Kit) *             text input OR select from inventory list
      If typed and not found: "إضافة كيت جديد للمخزون؟" prompt appears
    الكمية المتوفرة الحالية *     number input — initial stock
    وحدة الكمية                   dropdown: [وحدة] [علبة] [كيس] [لتر] [مل]
    حد التنبيه (أقل كمية) *       number input — alert threshold

  TIMING section:
    وقت الإنجاز *                 number input + [دقيقة / ساعة / يوم] toggle

  NORMAL RANGES section (المعدلات الطبيعية):
    Repeatable rows (same as before)

  MODAL FOOTER: [حفظ الفحص] + [إلغاء]

━━━ WHEN EDITING AN EXISTING TEST ━━━
  Modal opens pre-filled with all existing data.
  User can change ANY field including:
    - السعر للمريض (price update)
    - سعر Lab to Lab (inter-lab price update)
  SPECIAL SECTION shown only in EDIT mode:

  "إضافة كمية للمخزون" card (below existing fields, in edit modal):
    ┌──────────────────────────────────────────────┐
    │ 📦 إضافة كمية جديدة للكيت                   │
    │ الكمية الحالية: 12 وحدة                      │
    │ الكمية المُضافة: [____] وحدة *               │
    │ ملاحظة: سيُسجَّل كسحب من الجرار تلقائياً   │
    │   اسم الشركة (اختياري): [dropdown]           │
    │   [+ تسجيل الإضافة]  secondary button        │
    └──────────────────────────────────────────────┘
  On save quantity addition:
    Kit stock increases
    Drawer withdrawal auto-created (category: مشتريات) IF company entered
    Toast: "تمت إضافة [X] وحدة لكيت [الاسم]"
    Modal stays open so user can also edit other fields

━━━ TESTS TABLE — NEW COLUMNS ━━━
  # | اسم الفحص | الرمز | الفئة | السعر للمريض (₪) | سعر L2L (₪) | الكيت | الكمية | إجراءات

  The "الكمية" column:
    Shows current kit stock
    Color: green if above threshold | orange if at/near threshold | red if 0

  "سعر L2L" column header tooltip: "سعر Lab to Lab — للإرسال لمختبر خارجي"
