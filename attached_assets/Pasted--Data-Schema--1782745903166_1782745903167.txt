تحديث هيكل البيانات في الذاكرة (Data Schema)
يتم إضافة الجداول/المصفوفات التالية إلى قاعدة البيانات المحلية (⁠localStorage⁠) لربطها بكل قسم بدقة:
// واجهات البيانات (Interfaces)
interface DrawerCash {
  id: string;
  department: string; // اسم القسم (عيادة، مختبر، إلخ)
  date: string;
  openingBalance: number; // الكاش في الجرار قبل بدء العمل
}

interface Expense {
  id: string;
  department: string;
  date: string;
  amount: number;
  description: string;
  type: 'PERSONAL' | 'PURCHASE'; // التفرقة بين الشخصي والمشتريات
}

2. واجهة المستخدم الفرعية لكل قسم (UI Components)
في كل صفحة من صفحات الأقسام، سيتم عرض لوحة تحكم مالية مصغرة أسفل الشاشة تحتوي على 3 بطاقات (Cards) تفاعلية:
 البطاقة الأولى: 💰 الجرار (صندوق بداية اليوم)
 الوظيفة: إدخال المبلغ الموجود في الدرج صباحاً قبل استقبال أي مريض.
 الواجهة: حقل إدخال رقمي يعرض "رصيد الجرار الافتتاحي"، مع زر "اعتماد". بمجرد اعتماده، يثبت هذا الرقم كقاعدة أساسية لحسابات اليوم للقسم.
 البطاقة الثانية: ☕ المصروفات الشخصية
 الوظيفة: تسجيل أي مبالغ تُسحب لأغراض غير تشغيلية (طعام، دخان، احتياجات شخصية).
 الواجهة: نموذج إدخال سريع يحتوي على (وصف السحب + المبلغ) مع زر "سحب من الجرار". تظهر أسفله قائمة مصغرة بمسحوبات اليوم ليتمكن الموظف من مراجعتها.
 البطاقة الثالثة: 🛒 المشتريات التشغيلية
 الوظيفة: تسجيل المستهلكات المشتراة خصيصاً للقسم (سرنجات للعيادة، كتات للمختبر، منظفات وورق للأشعة).
 الواجهة: نموذج إدخال يحتوي على (اسم المادة المشتراة + المبلغ) مع زر "تسجيل مشتريات". يُربط هذا القسم لاحقاً بنظام إدارة المخزون لخصم الكميات إن لزم الأمر.
3. التحديث على المعادلات المالية (Financial Logic)
لضمان دقة التقارير في نهاية اليوم وعرضها للمالك، تتغير معادلة "الكاش الفعلي الموجود في الدرج" (Actual Cash in Drawer) لكل قسم لتصبح كالتالي:

الكاش الفعلي في الجرار عند الإغلاق = (رصيد الجرار الافتتاحي) + (إجمالي المدفوعات المستلمة من المرضى في هذا القسم) - (إجمالي المصروفات الشخصية) - (إجمالي المشتريات).

مثال تطبيقي (قسم المختبر):
 الجرار الافتتاحي: 200 ₪
 إيرادات المختبر اليوم: 500 ₪
 مصاريف شخصية (طعام): 30 ₪
 مشتريات القسم (كحول طبي وقطن): 70 ₪
 الكاش الفعلي الواجب تسليمه: (200 + 500) - (30 + 70) = 600 ₪.
4. الترحيل إلى النظام المالي المركزي (Super Admin Dashboard)
 عزل الأرباح: عند احتساب "صافي الأرباح" للمركز ككل في نهاية الشهر، سيقوم النظام تلقائياً بجمع (مشتريات الأقسام) واعتبارها "مصاريف تشغيلية مقبولة" تُخصم من الإيراد للوصول للربح التشغيلي، بينما يقوم بعزل (المصروفات الشخصية) كـ "مسحوبات مالك" لكي لا تؤثر على حساب التكلفة التشغيلية الحقيقية للمركز الطبي.
 التزامن السحابي: يتم تخزين هذه القيود الثلاثة لحظياً في الذاكرة المحلية، وتُرحل كأعمدة منفصلة وواضحة عند إجراء التزامن مع ⁠Google Sheets⁠ لتسهيل التدقيق المحاسبي ومراقبة حركة كل قسم على حدة.
